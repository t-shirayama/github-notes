## 危険度の見方

| ラベル | 意味                                                   |
| ------ | ------------------------------------------------------ |
| `安全` | 履歴を書き換えず、共有ブランチでも比較的使いやすい操作 |
| `注意` | 作業中の変更やコミットの状態を理解してから使う操作     |
| `危険` | 履歴を書き換える、または変更を失う可能性がある操作     |

## 基本方針

- ブランチ移動は `git switch` を使う。
- ファイルを戻すときは `git restore` を使う。
- 古い記事や特殊な操作では `git checkout` も出てくるので、違いは [ブランチ操作](./コマンド/ブランチ操作.md#checkout-と-switch-の違い) を参照する。
- push済みの修正を取り消す場合は、原則 `git revert` で履歴を残す。
- 履歴を書き換える操作は、個人ブランチかつ影響範囲を確認してから行う。
