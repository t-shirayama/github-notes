# 困ったとき

Gitで迷ったとき、危険な操作を打つ前に見るメモです。

## まず打つ

`安全`

### いつ使う

今の状態が分からない、戻し方が分からない、危険な操作を打つ前に状況を整理したいとき。

### コマンド例

```sh
# 今いるブランチ、変更ファイル、ステージ状態を確認
git status

# 最近の履歴とブランチの関係を確認
git log --oneline --graph --decorate --all -n 20

# HEADやブランチが過去にどこを指していたか確認
git reflog --date=local -n 20
```

### 注意点

この3つは確認用なので安全です。出力を見ても判断できない場合は、`reset --hard` や `push --force` を打つ前に止まります。

### 戻し方があるか

確認だけなので戻し方は不要です。

## やってはいけないこと

`危険`

### いつ使う

事故対応中に、さらに状況を悪くしないための確認です。

### コマンド例

```sh
# すぐには打たない
git reset --hard
git push --force
git clean -fd
```

### 注意点

- よく分からないまま `git reset --hard` しない。
- よく分からないまま `git push --force` しない。
- 未commitの変更がある状態で危険コマンドを打たない。
- `git clean -fd` の前に必ず `git clean -nd` で削除対象を確認する。
- push済みの修正を取り消すだけなら、まず `git revert` を検討する。

### 戻し方があるか

commit済みの履歴は `git reflog` で戻せることがあります。未commitの変更やGit管理外ファイルは戻せないことが多いです。
