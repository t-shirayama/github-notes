# 差がつくコマンド

日常操作に慣れてきたら覚えたい、調査・レビュー・事故対応で役立つコマンドです。

## 実際に試す

コマンドの動きは、練習用リポジトリを作って試すと理解しやすいです。この備忘録リポジトリ本体ではなく、[ハンズオン](./ハンズオン/README.md) の手順で作成した練習用リポジトリで実行します。

| コマンド | ハンズオン |
| -------- | -------------- |
| `reflog` | [見えなくしたcommitを探して復元する](./ハンズオン/reflog.md) |
| `cherry-pick` | [別ブランチの1commitだけ取り込む](./ハンズオン/cherry-pick.md) |
| `bisect` | [バグを入れたcommitを探す](./ハンズオン/bisect.md) |
| `blame` | [行ごとの変更commitを追う](./ハンズオン/blame.md) |
| `worktree` | [別ブランチを別ディレクトリで開く](./ハンズオン/worktree.md) |
| `range-diff` | [履歴整理前後のcommit列を比較する](./ハンズオン/range-diff.md) |
| `clean` | [Git管理外ファイルの削除対象を確認する](./ハンズオン/clean.md) |

## reflog で過去の移動履歴を見る

`安全`

### いつ使う

reset、rebase、amendなどで見えなくなったcommitを探したいとき。

### コマンド例

```sh
git reflog
git reflog --date=local
```

### 注意点

`reflog` はローカルリポジトリでHEADやブランチがどこを指していたかを記録しています。リモートの共有履歴そのものではありません。

### 戻し方があるか

戻したいcommitが見つかったら、ブランチを作って保護できます。

```sh
git branch recover-branch <commit>
```

## cherry-pick で特定commitだけ取り込む

`注意`

### いつ使う

別ブランチの特定commitだけを今のブランチへ取り込みたいとき。

### コマンド例

```sh
git cherry-pick <commit>
```

### 注意点

同じ変更を別commitとして取り込むため、後でmergeすると重複やコンフリクトの原因になることがあります。

### 戻し方があるか

cherry-pick中なら中止できます。

```sh
git cherry-pick --abort
```

commit済みなら `git revert <cherry-pickしたcommit>` で取り消せます。

## bisect でバグを入れたcommitを探す

`安全`

### いつ使う

「昔は動いていたが、どこかのcommitから壊れた」という原因commitを探したいとき。

### コマンド例

```sh
git bisect start
git bisect bad
git bisect good <動いていたcommit>

# Gitが示すcommitでテストして結果を登録
git bisect good
git bisect bad

# 終わったら元に戻る
git bisect reset
```

### 注意点

各commitでテストできる状態が必要です。自動テストがあると特に強力です。

### 戻し方があるか

調査を終えたら必ず `git bisect reset` します。

## blame で行ごとの変更者とcommitを見る

`安全`

### いつ使う

ある行がいつ、どのcommitで変更されたか知りたいとき。

### コマンド例

```sh
git blame <file>
git blame -L 10,30 <file>
```

### 注意点

犯人探しではなく、背景を追うために使います。commitを見つけたら `git show <commit>` で意図を確認します。

### 戻し方があるか

確認だけなので戻し方は不要です。

## worktree で別ブランチを同時に開く

`注意`

### いつ使う

今の作業を残したまま、別ブランチを別ディレクトリで確認・修正したいとき。

### コマンド例

```sh
git worktree add ../git-memo-main main
git worktree list
git worktree remove ../git-memo-main
```

### 注意点

同じブランチを複数のworktreeで同時にcheckoutすることはできません。作業ディレクトリが増えるので、不要になったら削除します。

### 戻し方があるか

不要になったworktreeは `git worktree remove <path>` で外します。

## range-diff でrebase前後を比較する

`安全`

### いつ使う

rebaseや履歴整理の前後で、commit列がどう変わったか確認したいとき。

### コマンド例

```sh
git range-diff main..feature-before main..feature-after
```

### 注意点

PRレビューで「rebase後に意図しない変更が混ざっていないか」を確認するのに便利です。

### 戻し方があるか

比較だけなので戻し方は不要です。

## clean -n で不要ファイル削除を事前確認する

`危険`

### いつ使う

Git管理外の生成物や一時ファイルを削除したいとき。

### コマンド例

```sh
# 削除対象を確認するだけ
git clean -n

# ディレクトリも含めて確認するだけ
git clean -nd

# 実際に削除する
git clean -fd
```

### 注意点

`git clean -fd` はGit管理外ファイルを削除します。戻せないことが多いので、必ず `-n` で対象を確認してから実行します。

### 戻し方があるか

Git管理外のファイルは基本的に戻せません。必要なファイルは先に退避します。
